# Builder Roadmap — outbox.cafe

The improvement log for the cafe: what shipped, what's queued, what was decided
against and why. The recurring `outbox-cafe-builder` scheduled task (Mini,
Tue+Fri 1pm) reads this before building and appends a batch entry after every
ship. Lives in the repo (not Claude project memory) so the Mini's updates
survive — the laptop pushes its memory snapshot over the Mini's hourly, which
would clobber Mini-side memory edits. Moved here 2026-06-10.

Standing decisions: engagement stays **fully auto** (no human review queue, no second-opinion model gate) — the whole thing is low-stakes, light-hearted, fun. Everything haiku→opus. Social posters are ephemeral (don't keep); on-site images (thumbnails) → Vercel Blob. The "Tailscale fire-a-gen button" was never built — cut, don't build.

**Batch 10 — SHIPPED (2026-06-16, commit eab71ff, builder run):** the cabinet gets a finder. The collection had grown to 239 cards (4/day, ~+120/month) with only STUMBLE (random) and BINDER (kept) as controls — an undifferentiated wall and no way to find a card. Added a live client-side search to `rebuild_cabinet()` in generate.py: each `.card-wrap` now carries a `data-search` attr built from title + `era` + `format` + `tone` (the three spec dims that were already computed per card but never rendered — this resurrects that dead computation), lowercased + attribute-escaped. A `.finder` input under the hero meta-bar filters cards live (all whitespace terms must match, AND), shows an `N / total` count, and a "nothing matches" message; combines with binder-only purely via CSS (both add `display:none` → intersection). Zero backend, no churn, mobile-responsive (`width: min(420px, 86vw)`, flex-wrap). On-brand zine styling (paper/ink input, dim shadow → gold on focus). Verified: compiles both pythons; standalone `rebuild_cabinet()` emits 239 data-search attrs; browser-tested via Preview MCP — "" → 239, "puzzle" → 9, "1962" (era) → 9, multi-term "the forward" → 1, "zzxqq" → 0 + empty msg shown; live on https://outbox.cafe/archive/ (239 attrs deployed). Note: search matches the _condensed_ dim text (32/38/28-char truncations, some with "…" tails) — good enough for "puzzle"/"diner"/"1978"-style queries; full prose dims live in the page `<meta>` if a future richer filter ever wants them. (`.claude/launch.json` cabinet-static preview config added — gitignored, won't deploy.)
